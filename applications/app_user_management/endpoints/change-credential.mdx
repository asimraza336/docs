# Reset Credential API

This document describes the API endpoint `ResetCredentialView`, which handles the resetting of a user's credential (username or password).

## Endpoint

| Method | Path                             | Description                                       |
|--------|----------------------------------|---------------------------------------------------|
| POST   | `/api/appUser/reset-credential/` | Resets a user's credential (username or password). |

## 1. POST /api/appUser/reset-credential/

### Description

This endpoint resets a user's credential (either username or password) based on the provided details.

### Request Method

`POST`

### Path

`/api/appUser/reset-credential/`

### Location in Django Files

`app_name/views.py`

### Authentication

This endpoint does not require prior authentication.

### Permissions

None

### Request Structure

#### Headers

| Header         | Necessity | Description                         |
|----------------|-----------|-------------------------------------|
| `Content-Type` | Required  | `application/json`                  |

#### Body Parameters

| Parameter    | Necessity | Type    | Description                                                             |
|--------------|-----------|---------|-------------------------------------------------------------------------|
| `profile_pk` | Required  | Integer | The primary key of the `TempEmail` associated with the email.     |
| `type`       | Required  | String  | The type of credential being reset (either 'username' or 'password').   |
| `credential` | Required  | String  | The new credential value (either the new username or password).          |

### Response Structure

#### Success Response

**Status Code:** `200 OK`

| Field     | Type    | Description                                        |
|-----------|---------|----------------------------------------------------|
| `service` | String  | A message indicating that the credential was updated. |

### Error Handling

This endpoint does not have any specific error handling mechanisms beyond the standard Django error responses.

### Notes

- The `type` parameter determines whether the `credential` is treated as a username or password.
- This endpoint should be used when the user needs to reset their credential using a provided one-time password (OTP).
